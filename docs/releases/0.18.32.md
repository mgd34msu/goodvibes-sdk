# 0.18.32

`0.18.32` adds port-in-use checking to `DaemonServer` and `HttpListener` so hosts get a clear error instead of an unhandled `EADDRINUSE` crash, and changes the default daemon port from `3000` to `3141` to avoid conflicts with common development servers.

## What Changed

- `DaemonServer` and `HttpListener` now verify the target port is free before binding — a clear error message is emitted instead of crashing with `EADDRINUSE`
- Default daemon port changed from `3000` to `3141` to avoid conflicts with common local dev servers

## Why This Patch Exists

Port 3000 is the default for a large number of development frameworks (Next.js, Create React App, Rails, etc.), which meant the GoodVibes daemon would frequently crash on startup for developers running any of those servers alongside the platform. The new default port `3141` (a nod to π × 1000) is far less likely to collide. The pre-bind port check gives a clear, actionable error message for the remaining cases where the chosen port is already occupied.
