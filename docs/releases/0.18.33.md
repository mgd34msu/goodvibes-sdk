# 0.18.33 — SDK documentation + companion auth field

`0.18.33` ships the first comprehensive SDK documentation set and backfills release notes for 0.18.29 through 0.18.32.

## Summary

This release is documentation-only (plus one minor API addition). It adds the SDK architecture overview, security guide, performance tuning guide, observability reference, migration guide, and companion app pairing guide — content that was missing since the SDK began accumulating significant surface area. A `password` field is added to `CompanionConnectionInfo` so companion apps can authenticate with password-protected daemon endpoints.

## Changes

- Added `docs/architecture.md` — SDK layering, host-boundary model, package structure
- Added `docs/security.md` — auth model, TLS configuration, token rotation, secrets management
- Added `docs/performance.md` — connection pooling, event batching, memory management
- Added `docs/observability.md` — runtime event bus, health monitoring, diagnostic snapshots
- Added `docs/migration.md` — upgrade guide for 0.18.29 breaking changes
- Added `docs/pairing.md` — companion app pairing flow, QR code generation, token lifecycle
- Added release notes `docs/releases/0.18.29.md` through `docs/releases/0.18.32.md`
- Updated root README with pairing module, port-checking behavior, and new config entries
- Added `password` field to `CompanionConnectionInfo` so companion clients can authenticate with bearer-token-protected daemon endpoints

## Breaking behavior

None. The `password` field on `CompanionConnectionInfo` is optional.
