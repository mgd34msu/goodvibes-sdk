# 0.18.48 — Hot-patch: wire CompanionChatManager into daemon router

**Released**: 2026-04-16
**Type**: Bug fix (production 404)

## Summary

Fixes a production 404 on `POST /api/companion/chat/sessions` that external app consumers
hit against the live daemon. The companion-chat handler code shipped in 0.18.43, but the
manager was never constructed or injected — so the route guard was always falsy.

## Root Cause

`DaemonHttpRouter` gates companion chat routes on `companionChatManager` being present in
its context. `facade-composition.ts` never constructed or injected a `CompanionChatManager`,
so the route guard evaluated falsy on every request, returning 404 to all companion-chat
consumers.

## Fix

- `facade-composition.ts` now constructs `CompanionChatManager` and passes it through the
  daemon context so the route guard evaluates truthy.
- No API contract changes. No migration required.

## Impact

- Companion chat endpoints (`POST /api/companion/chat/sessions`, `POST /api/companion/chat/sessions/:id/message`, `GET /api/companion/chat/sessions/:id/events`) now function correctly against the live daemon.
- Existing operator and peer API surfaces unaffected.
