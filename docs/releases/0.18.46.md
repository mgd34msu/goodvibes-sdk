# 0.18.46 — CI build fix for RuntimeEventRecord variance

**Released**: 2026-04-16
**Type**: Bug fix (CI build)

## Summary

`0.18.45` broke the CI build (`bun run build` → `tsc -b` project references) with TS2322
variance errors. Local `bunx tsc --noEmit` did not surface the issue — only the
project-reference build catches it. This patch widens two connector return types to
resolve the variance mismatch.

## Root cause

`createEventSourceConnector` and `createWebSocketConnector` in
`transport-realtime/runtime-events.ts` declared their return type as
`DomainEventConnector<RuntimeEventDomain, RuntimeEventRecord>`, using the file-local
private bound `{readonly type: string}`. TypeScript inferred
`RemoteRuntimeEvents<{type:string}>`, which is invariant in the generic parameter and
therefore not assignable to `RemoteRuntimeEvents<AnyRuntimeEvent>`, producing:

```
Type 'RemoteRuntimeEvents<RuntimeEventRecord>' is not assignable to
type 'RemoteRuntimeEvents<AnyRuntimeEvent>'.
```

The error only appeared under `tsc -b` (project references) used by `bun run build`,
not under `tsc --noEmit` used by the local typecheck shortcut.

## Fix

Widened both connector factory return types from
`DomainEventConnector<RuntimeEventDomain, RuntimeEventRecord>` to
`DomainEventConnector<RuntimeEventDomain, AnyRuntimeEvent>`. Internal envelope casts
now use `SerializedRuntimeEnvelope<AnyRuntimeEvent>` to match. The local
`{readonly type: string}` bound on `SerializedRuntimeEnvelope<T>` and
`RemoteRuntimeEvents<T>` generic constraints is preserved — only the return-type
annotations changed.

## Companion file note

The upstream copy at `packages/transport-realtime/src/runtime-events.ts` still uses
the old signatures. That package does not import `AnyRuntimeEvent` (no import path at
that layer). The sync script `bun run sync:internal` is not wired into CI, so
downstream drift is tolerated under the existing precedent (see
`daemon-error-contract.ts`). Running sync would revert this fix — manual re-fix
required if invoked.

## Changes

- `packages/sdk/src/_internal/transport-realtime/runtime-events.ts` — widened
  `createEventSourceConnector` and `createWebSocketConnector` return types to
  `DomainEventConnector<RuntimeEventDomain, AnyRuntimeEvent>`
- No public API changes beyond what shipped in 0.18.45

## Breaking behavior

None. The public shape of `RuntimeEventRecord` and `AnyRuntimeEvent` is identical to
0.18.45.

## Migration

No migration steps beyond 0.18.45. See
