# 0.18.49 — DX Wave 1: transport errors, JSDoc, docs, examples

**Released**: 2026-04-16
**Type**: Enhancement (DX)

## Summary

Five concurrent WRFC chains shipped bundled, all passing 10.0/10.0. Addresses the
highest-leverage findings from the end-to-end DX deep-dive audit.

## Changes

### Transport error class (DX audit top leverage point)

`createTransportError` and `createNetworkTransportError` now throw `HttpStatusError` so
`error instanceof HttpStatusError` works as the error-handling documentation has always
claimed. The error category payload shape is finalized: `statusCode`, `method`, `url`,
and `requestId` fields.

### JSDoc coverage wave 1

All public transport types, operator client methods, and peer client methods now carry
structured JSDoc with `@param`, `@returns`, `@throws`, and `@example` annotations.

### Docs and examples

New `docs/transport-errors.md` guide covering error hierarchy, `instanceof` narrowing,
and category-based branching. Runnable code examples added to `docs/examples/`.

### Remaining DX audit items

- `AuthTokenInput` type alias exported from `@pellux/goodvibes-sdk/transport-http`.
- `listModels()` return type tightened to `readonly Provider[]`.

## Migration

No breaking changes. The `HttpStatusError` throw is a correctness fix — callers that
already catch `Error` continue to work; callers that use `instanceof HttpStatusError`
now get the expected behavior.
