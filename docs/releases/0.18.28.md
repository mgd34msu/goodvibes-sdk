# 0.18.28

`0.18.28` fixes the session-persistence boundary so hosts that set an explicit `surfaceRoot` are no longer split between correctly scoped session files and incorrectly shared pointer or recovery files.

## What Changed

- `platform/runtime/session-persistence` now threads `surfaceRoot` through:
  - `getUserSessionsDir(...)`
  - `getLastSessionPointerPath(...)`
  - `getRecoveryFilePath(...)`
  - pointer writes and reads
  - recovery writes, reads, checks, and deletes
- Hosts that save sessions under a scoped root now keep:
  - session files
  - last-session pointers
  - crash-recovery files
  under the same host-owned surface tree.

## Why This Patch Exists

The TUI was already saving session bodies through a `SessionManager` with `surfaceRoot: 'tui'`, but the shared pointer and recovery helpers were still resolving through the unscoped root. That left one part of the session surface on `.goodvibes/tui/...` and another part on the shared `.goodvibes/...` tree, which is the wrong boundary.

This patch makes the SDK respect the host-owned storage policy consistently.
