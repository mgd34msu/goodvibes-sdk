# 0.18.51 — RN crypto hotpatch: delete node adapter, async-cascade, atob guard

**Released**: 2026-04-16
**Type**: Critical fix + Breaking (pre-1.0 minor bump)

## Summary

This release closes the React Native bundling regression introduced by the 0.18.50 auth
barrel split. Three functions changed from sync to async as part of the Web Crypto
migration. Node 18 support is dropped (EOL April 2025).

> **TUI upgraders**: goodvibes-tui v0.18.27 is REQUIRED alongside this SDK release.
> Eight call sites in the TUI must be updated to `await` the newly async functions.
> Do NOT upgrade the SDK without upgrading the TUI in the same deploy.

## Changes

### CRITICAL — Orphaned `crypto-adapter.node.ts` deleted (Route B)

`packages/sdk/src/_internal/platform/runtime/auth/crypto-adapter.node.ts` is deleted.
Nothing routed to it; Node >= 19 exposes `globalThis.crypto` natively. The `node:crypto`
fallback is no longer needed. Node 18 reached EOL in April 2025 and is not supported.

The main `crypto-adapter.ts` comment referencing the node adapter as an "explicit Node
escape hatch" is removed to match.

### BREAKING — Three functions changed from sync to async

These functions now return `Promise` and must be `await`-ed by all callers:

| Function | Previous | Now |
|---|---|---|
| `OAuthClient.beginAuthorization` | sync | `async` |
| `beginOpenAICodexLogin` | sync | `async` |
| `SubscriptionManager.beginOAuthLogin` | sync | `async` |

**Migration**:
```ts
// Before:
const { authorizationUrl } = client.beginAuthorization(params);

// After:
const { authorizationUrl } = await client.beginAuthorization(params);
```

Callers that do not `await` will receive a `Promise<OAuthStartState>` instead of an `OAuthStartState` and
will silently break at runtime. TypeScript will surface this as a type error if the
return value is used.

### MINOR — `decodeJwtPayload` atob guard

`decodeJwtPayload` in `oauth-core.ts` now throws at the top of the function if
`typeof atob !== 'function'`:

```ts
if (typeof atob !== 'function') {
  throw new Error('decodeJwtPayload requires a global atob(); available in browsers, RN Hermes, and Node >= 16');
}
```

The surrounding `try/catch` still returns `null` for malformed JWTs. The throw only fires
when the runtime is genuinely incompatible (no `atob` at all), not for decode errors.

### MINOR — Minimum Node engine raised to 19

`packages/sdk/package.json` now declares `"engines": { "node": ">=19" }`. Node 18 is EOL
and is not supported. Node 20 LTS and 22 LTS are the supported Node targets.

### CI — `rn-bundle-node-imports.test.ts` hard-fails on missing dist

Previously the test skipped with a `console.warn` if a dist file did not exist, allowing
CI to pass silently when the build step was skipped. The test now calls
`expect(existsSync(filePath)).toBe(true)` and fails loudly.

## Migration Checklist

- [ ] Update goodvibes-tui to v0.18.27 (required — 8 async call sites)
- [ ] `await client.beginAuthorization(...)` at all call sites
- [ ] `await beginOpenAICodexLogin(...)` at all call sites  
- [ ] `await manager.beginOAuthLogin(...)` at all call sites
- [ ] Remove any `OAuthClient` imports from `@pellux/goodvibes-sdk/auth` (use `./oauth`)
- [ ] Confirm Node >= 19 in CI and production environments

## Out of Scope

The 8 TUI call sites (`goodvibes-tui`) that reference the newly async functions are handled
in the companion TUI patch (v0.18.27). They are not modified in this SDK release.
