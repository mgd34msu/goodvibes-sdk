# 0.18.15

`0.18.15` is the first patch release after the full `0.18.14` SDK launch, and it finishes the pending platform extraction work that was still sitting in the repo instead of the published package.

This release is not a packaging-only patch. It turns the extracted platform implementation into the canonical shipped SDK state and removes several remaining “copied from the app” seams inside the runtime support layer.

## What This Release Finishes

The SDK now carries the pending extracted platform implementation that had been staged locally but not yet shipped in the canonical release line.

That expanded internal platform tree now includes the extracted implementation for:

- ACP connection and manager runtime
- adapter surfaces for external chat and notification systems
- agent orchestration, session, worktree, and WRFC runtime modules
- automation manager, delivery, execution, reconcile, scheduling, and sync modules
- channel runtime, delivery routing, reply pipeline, surface registry, and plugin registry modules
- config managers, helper-model wiring, secrets, subscriptions, and service registries
- control-plane approval, gateway, session-broker, operator-contract, method-catalog, and schema modules
- daemon host composition, CLI, service manager, server, router, listener, and transport-event modules
- conversation/orchestrator/compaction runtime modules
- hooks dispatcher, hook API, runners, and workbench modules
- knowledge ingest, compilation, lint, service, scheduling, packet, and API modules
- MCP client, registry, and API modules
- media and multimodal runtime modules
- runtime sandbox, telemetry, tasks, tools, remote runtime, settings, store, and state helpers
- web-search providers and service modules
- watcher registry and store modules

The package model is unchanged:

- one public npm package
- one canonical package name: `@pellux/goodvibes-sdk`
- stable subpath exports
- no public multi-package install story

## SDK-Native Runtime Refactor

This release also cleans up the runtime support layer so it behaves like SDK-owned source rather than a set of package-self-importing carryovers.

Key refactors in this patch:

- The public `@pellux/goodvibes-sdk/platform/runtime/ui/provider-health/*` path is now the source-owned implementation path.
- The older `platform/runtime/provider-health/*` internal path is now compatibility-only and re-exports from the source-owned public runtime UI path.
- The copied runtime barrels and helper modules now import local SDK source directly instead of routing back through `@pellux/goodvibes-sdk/...` for their own implementation.
- The runtime host/bootstrap helper layer now exposes neutral host/runtime naming alongside compatibility aliases:
  - background provider discovery
  - background MCP discovery
  - runtime system-prompt loading
  - runtime model restore
  - host runtime event registration
  - host service startup

That keeps compatibility for existing internal callers while making the underlying code read like SDK host/runtime infrastructure instead of extracted TUI scaffolding.

## Release And Verification Changes

This release also tightens the release path itself.

Included release-path changes:

- GitHub Releases now prefer dedicated `docs/releases/<version>.md` documents when present instead of falling straight back to a changelog excerpt
- pack/install/release helpers now create their temp staging areas under repo-local `.tmp/` paths
- the SDK repo now ignores `.tmp/` so validation and dry runs do not leave untracked temp artifacts in the worktree

This matters because the SDK release process now has:

- a real long-form release document
- reproducible temp staging under the repo root
- clean worktrees after release validation
- a stable release-note source for GitHub releases

After `0.18.15`, the canonical SDK release line includes the pending extracted platform implementation instead of leaving that work stranded as a local-only tree.

For consumers, the headline is simple:
