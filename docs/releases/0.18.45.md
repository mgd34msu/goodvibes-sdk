# 0.18.45 — ProviderRegistry canonical accessors + RuntimeEventRecord discriminated union

`0.18.45` tightens the `ProviderRegistry` access API and makes `RuntimeEventRecord` a fully typed discriminated union.

## Summary

Two related type-safety improvements ship together. `ProviderRegistry` gains explicit accessor methods that make intent clear at every call site: `has()` for existence checks, `require()` for guaranteed access, and `tryGet()` for nullable access. The existing `get()` is preserved for backward compatibility but marked `@deprecated`. Separately, `RuntimeEventRecord` becomes a discriminated union over all known event types — consumers can now narrow events by switching on `event.type` instead of casting `event.payload`.

## Changes

### ProviderRegistry canonical accessors

Three new methods on `ProviderRegistry`:

- **`has(id: string): boolean`** — returns `true` if a provider with the given ID is registered. Non-throwing.
- **`require(id: string): Provider`** — returns the provider or throws `ProviderNotFoundError` if absent. Use this as the throwing accessor going forward.
- **`tryGet(id: string): Provider | undefined`** — returns the provider or `undefined`. Use this as the safe nullable accessor.
- **`get(id: string): Provider`** — still throwing, now marked `@deprecated`. Migrate callers to `require()` (same semantics) at your own pace.

### RuntimeEventRecord discriminated union

`RuntimeEventRecord` is now a union alias over all concrete event types (`AnyRuntimeEvent`). Each member carries a unique `type` literal, enabling exhaustive narrowing:

```ts
switch (event.type) {
  case 'agent.completed':
    // event.payload is AgentCompletedPayload — no cast needed
    break;
  case 'session.closed':
    // event.payload is SessionClosedPayload — no cast needed
    break;
}
```

Consumers that previously wrote `event.payload as SomeType` can remove the cast after switching on `event.type`.

## Breaking behavior

None. All changes are additive or deprecation-only. `get()` still works; `RuntimeEventRecord` usage that does not narrow continues to compile.
