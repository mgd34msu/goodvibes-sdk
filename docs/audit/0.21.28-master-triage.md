### Top 7 issues the user must see first (most severe under enterprise bar)

1. **SEC-04** — Postinstall tarball fetched without integrity verification (CRITICAL, supply-chain). Silent registry-substitution → code execution in every downstream consumer. Rubric escalation: unsigned postinstall fetch = CRITICAL.
2. **SEC-10** — `Math.random()` used to generate bytes in `transport-http/http-core.ts:52`. Rubric escalation: crypto misuse = CRITICAL even if call-site non-auth, because auditors grep for it and fail vendor review on sight.
3. **SEC-01** — Auth-user store (scrypt password hashes) written world-readable (0644). Offline brute-force vector; violates "0600 for identity state" invariant that already governs adjacent files.
4. **SEC-03** — `/login` bypasses rate limiter + origin check. Unbounded scrypt-cost-throttled online brute-force; network-bound when `hostMode=network`. Rubric: auth bypass = CRITICAL.
5. **SEC-02** — Daemon-home migration propagates 0644 mode from legacy TUI files into new canonical location — identity state leaks through migration.
6. **OBS-01 + OBS-02 + OBS-03 + OBS-04 (audit trail)** — HTTP requests, auth events, outbound provider/LLM calls produce **zero** access-log events. No SIEM feed possible. Compliance-adjacent audit-trail gap = HIGH minimum.
7. **PERF-01 + SEC-06 + PERF-09** — Unbounded Map/audit-array growth in SessionBroker, CompanionChatRateLimiter, DistributedRuntime audit. Long-running daemons leak linearly. Enterprise bar: unbounded memory growth = HIGH minimum.

### Verdict

**This SDK is NOT enterprise-adoptable today.** Gates that must close before vendor-security sign-off:

- Crypto misuse (`Math.random` bytes, SEC-10) — deferred to Wave 5 per user decision (current use appears to be client-side request ID generation, not security-sensitive entropy; however auditors grep for `Math.random` and will flag it regardless)
- Supply-chain integrity (postinstall SHA-pin, SEC-04) — **blocker**
- Identity-file permissions (SEC-01, SEC-02, SEC-12) — **blocker**
- `/login` rate-limit + origin check (SEC-03) — **blocker**
- HTTP access-log + auth-event emission (OBS-01, OBS-02) — **blocker** (SOC2 audit-trail)
- Body-size caps on all state-changing routes (SEC-05) — **blocker** (DoS)
- Unbounded Maps documented + bounded (PERF-01, SEC-06, PERF-09) — **blocker**
- Prompt/response redaction policy (OBS-05, OBS-06, OBS-24) — **blocker** (privacy)

Waves 1–3 (SDK 0.21.29 → 0.21.31) close every enterprise-blocker above. Waves 4–5 are quality/architecture polish.

## 2. Rescoring rubric (applied verbatim)

Under the enterprise-adoption bar, the following auto-escalations apply. Anything **not** in this list keeps its original severity.

- **Supply-chain finding** (unpinned deps, unsigned postinstall fetches, integrity gaps) → HIGH minimum; CRITICAL if code-execution reachable.
- **Crypto misuse** (`Math.random` for security bytes, weak hash, hardcoded secret) → CRITICAL.
- **Unbounded memory growth** in long-running paths → HIGH minimum; CRITICAL if reachable without auth.
- **Privacy leak** (raw PII/secrets in logs/telemetry/metrics without redaction) → HIGH minimum.
- **Auth bypass / rate-limit bypass** → CRITICAL.
- **Audit-trail gap** (HTTP unlogged, LLM calls unrecorded, auth events unemitted) → HIGH minimum; CRITICAL if compliance-adjacent.
- **DoS vector** reachable without auth → HIGH minimum.
- **Insecure default config** (CORS:*, no rate limit, debug-log-by-default) → HIGH minimum.
- **Unsafe type cast on untrusted input** → HIGH.
- **Sync I/O in HTTP request path** → HIGH if measurable latency, MEDIUM otherwise.
