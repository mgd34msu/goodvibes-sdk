# 0.18.50 — Tier 4 Wave B + DX Wave 2 (errors, Orchestrator options, auth split, forSession, @see)

**Released**: 2026-04-16
**Type**: Breaking (pre-1.0 minor bump per versioning policy)

## Summary

Five concurrent WRFC chains, all passing 10.0/10.0. Biggest breaking surface change since
the 0.18.29 boundary separation, shipped as a minor bump per the pre-1.0 versioning policy.

## Changes

### Unified error hierarchy with tagged kind (Tier 4 #1)

New `SDKErrorKind` union: `auth | config | contract | network | not-found | rate-limit | server | validation | unknown`. `GoodVibesSdkError` now carries a typed `kind` discriminant derived from `category` at construction. All SDK-thrown errors map to exactly one kind. Existing `HttpStatusError` / `ConfigurationError` / `ContractError` preserved as deprecated subclasses so `instanceof` still works.

```ts
try {
  await client.getSession(id);
} catch (err) {
  if (err instanceof GoodVibesSdkError && err.kind === 'not-found') {
    // handle 404
  }
}
```

### Orchestrator options-object constructor (Tier 4 #3) — **BREAKING**

`Orchestrator` constructor changed from 11 positional arguments to a single `OrchestratorOptions`
object. Optional fields carry sensible defaults via destructuring; required fields are validated.

```ts
// Before
new Orchestrator(conversation, getViewportHeight, scrollToEnd, toolRegistry, permissionManager, getSystemPrompt, hookDispatcher, flagManager, requestRender, runtimeBus, services);

// After
new Orchestrator({
  conversation, getViewportHeight, scrollToEnd, toolRegistry, permissionManager, services,
  // optional: getSystemPrompt, hookDispatcher, flagManager, requestRender, runtimeBus
});
```

All in-tree callsites (TUI production + tests) migrated in the same wave.

### Auth client split into focused responsibilities (Tier 4 #6)

Extracted four classes under `packages/sdk/src/_internal/platform/auth/`:

- `TokenStore` — persistence wrapper with `hasToken()` convenience
- `SessionManager` — login lifecycle + persist-on-login semantics
- `OAuthClient` — PKCE flow / code exchange / refresh / JWT decode
- `PermissionResolver` — role/scope checks over `ControlPlaneAuthSnapshot`

Existing `GoodVibesAuthClient` remains the unchanged thin facade over the split classes. Aggregated token methods (`getToken`/`setToken`/`clearToken`) are now `@deprecated` pointing at the focused classes. Non-breaking for existing consumers.

### realtime.forSession(sessionId) helper (R2.1)

New standalone helper on the realtime transport layer. Takes `(events, sessionId)` and returns a pre-filtered `DomainEvents` view where every `on`/`onEnvelope` callback fires only for events matching the given session. Eliminates the repeated manual filter pattern.

```ts
import { createNodeGoodVibesSdk, forSession } from '@pellux/goodvibes-sdk/node';

const sdk = createNodeGoodVibesSdk({ baseUrl, authToken });
const events = await sdk.realtime.viaSse();
const mine = forSession(events, sessionId);
mine.turn.onEnvelope('STREAM_DELTA', (env) => { /* only this session */ });
```

Re-exported from all 5 runtime entry-points (`/node`, `/browser`, `/web`, `/react-native`, `/expo`).

### DX Wave 2 — @see JSDoc tags (R6.1)

Public `GoodVibesSdk` namespaces (`operator`, `peer`, `auth`, `realtime`) + `createGoodVibesSdk` factory + auth-related `GoodVibesSdkOptions` fields carry `@see` pointers to the relevant docs. Absolute GitHub URLs so they resolve from the published npm tarball.

## Migration

- **Orchestrator constructor (BREAKING):** migrate from positional args to the `OrchestratorOptions` object form. See before/after above.
- **Error handling (additive):** replace `instanceof` chains with `err.kind ===` discriminant matching where it simplifies the code. Existing `instanceof HttpStatusError` etc. still works.
- **Auth classes (additive):** the four split classes are available alongside the existing facade. Prefer them for new code; the facade's aggregated token methods are deprecated but functional.
- **Realtime filtering (additive):** adopt `forSession(events, sessionId)` where panels/consumers previously filtered by sessionId manually.
