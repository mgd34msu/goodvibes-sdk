# 0.18.29

`0.18.29` completes a broad SDK/TUI boundary separation pass, removing terminal-specific types, TUI store domains, and hardcoded surface assumptions from the reusable platform surface, and adds several new platform capabilities that SDK consumers can use directly.

## What Changed

### Removed TUI-specific internals

- Deleted `types/grid.ts` (`Cell`, `Line`, `createEmptyLine`, `createStyledCell`) and `core/history.ts` (`InfiniteBuffer`) — orphaned TUI rendering modules with no SDK consumer use case
- Cleaned `BlockMeta` in `core/conversation.ts`: removed `blockIndex`, `startLine`, `lineCount`, `collapseKey` rendering fields; surfaces extend `BlockMeta` with their own rendering coordinates
- Removed unused `_getWidth` and `_configManager` constructor parameters from `ConversationManager`
- Decoupled `core/conversation-diff.ts` from `BlockMeta` by introducing a standalone `DiffParseResult` interface for `parseDiffForApply`
- Removed TUI-specific store domains from `RuntimeState`: replaced `panels: PanelDomainState` with `panels: Record<string, unknown>` and replaced `uiPerf: UiPerfDomainState` with `surfacePerf: SurfacePerfDomainState`
- Removed TUI-specific selectors (`selectPanels`, `selectActivePanels`, `selectFocusedPanel`) from SDK store selectors
- Split `utils/clipboard.ts`: removed the OSC 52 `copyToClipboard()` terminal write function from the SDK and added a `ClipboardWriteFunction` type so surfaces inject their own implementation; kept `pasteFromClipboard()` and `pasteImageFromClipboard()` in the SDK
- Removed `utils/splash-lines.ts` (ASCII art terminal banner) from the SDK platform surface
- Made the TUI surface config-driven in `channels/surface-registry.ts` instead of hardcoded as always-enabled

### New platform capabilities

- Added `undo()` and `redo()` methods to `ConversationManager` — previously TUI-only, now available as pure message-list operations
- Created `config/api-keys.ts` with `getConfiguredApiKeys()` and `resolveApiKeys()` containing the canonical 30+ provider-to-environment-variable mapping and three-tier key resolution (env vars, SecretsManager, skip)
- Extended the plugin loader with `additionalDirectories` and `entryDefault` options in `PluginPathOptions`

### Component health monitoring (renamed from Panel)

- Generalized health monitoring infrastructure from Panel-specific to Component-generic:
  - `PanelHealthMonitor` → `ComponentHealthMonitor`
  - `PanelResourceContract` → `ComponentResourceContract`
  - `PanelHealthState` → `ComponentHealthState`
  - `PanelResourceEntry` / `PanelResourceSnapshot` / `PanelConfig` → `ComponentResourceEntry` / `ComponentResourceSnapshot` / `ComponentConfig`
  - Old `Panel*` names preserved as deprecated backward-compatible aliases
- Updated `RuntimeServices.panelHealthMonitor` to `componentHealthMonitor: ComponentHealthMonitor` throughout the services layer

### Selector and service additions

- Added `selectSurfacePerf` and 8 missing platform domain selectors: `selectOrchestration`, `selectCommunication`, `selectAutomation`, `selectRoutes`, `selectControlPlane`, `selectDeliveries`, `selectWatchers`, `selectSurfaces`
- Created `runtime/service-queries.ts` as the canonical platform-named barrel for the existing `ui-service-queries.ts` query interfaces

### Terminology cleanup

- Replaced hardcoded TUI references in ops playbooks (`permission-deadlock.ts`: "Ink render tree" → "host render context") and session return context (`session-return-context.ts`: "terminal coding session" → "coding session")
- Updated bookmark manager docstring to use generic terminology instead of TUI-specific "collapse key"
- Documented platform vs surface classification in `utils/terminal-width.ts` and `utils/notify.ts` for future cleanup reference

## Why This Patch Exists

The SDK platform surface had accumulated a significant number of TUI-specific types, rendering fields, store domains, and hardcoded surface assumptions that no SDK consumer outside the TUI should ever need to depend on. This release removes those leaks systematically so downstream hosts can import and compose the platform surface without taking on TUI-internal rendering contracts or terminal-specific behavior.
