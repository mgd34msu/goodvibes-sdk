# 0.18.26

`0.18.26` fixes the SDK version fallback leak that was still embedding an old SDK version string into downstream host binaries.

The concrete problem was in the SDK’s own runtime version module:

- `packages/sdk/src/_internal/platform/version.ts` still had a baked fallback literal of `0.18.14`
- compiled hosts can embed that fallback string even when the package version has already moved forward
- that meant downstream binaries could carry an internal stale SDK version string long after the package itself had been patched

For `goodvibes-tui`, that created exactly the kind of version confusion that should not exist:

- the TUI app version was current
- but a stale SDK fallback string was still being bundled into the binary

## What Changed

### Build-Time Version Sync

The SDK now syncs its baked runtime version fallback from the root package version before every build.

That keeps:

- the root workspace version
- the SDK package version
- the baked fallback in `platform/version`

aligned as one release line.

### Regression Coverage

This release adds a regression test that reads the root package version and the SDK runtime version source and fails if they drift apart.

That protects the exact failure mode that let the stale fallback survive through release packaging.

### Documentation Cleanup

The stale static SDK-version lines in the top-level docs were removed.

Those lines had already drifted and were reinforcing the same maintenance problem: static version text in places that were not actually being kept in sync by the release path.

## Downstream Impact

This patch matters for any host that compiles or bundles the SDK into its own executable or application shell.

After this fix:

- the SDK no longer carries a stale baked version fallback from an older release
- downstream binaries stop embedding that old SDK version string by accident
- visible host version surfaces can stay tied to the actual host product version without conflicting embedded SDK release residue

## Validation

This release was validated with:

- `bun run sync:version`
- `bun run validate`
- `bun run release:dry-run`
- downstream rebuild confirmation from `goodvibes-tui` showing the stale embedded `0.18.14` leak is removed once the TUI updates to this SDK version
