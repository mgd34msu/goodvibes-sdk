# 0.18.38 — Wave 2 / Tier 1 hot-path performance caches

**Released**: 2026-04-16
**Type**: Patch (performance)

## Summary

Five targeted hot-path caches that eliminate repeated computation during turn loops and high-throughput SSE streaming.

## What changed

### P1: `ProviderRegistry.getModelRegistry()` rebuild cache

**File**: `packages/sdk/src/_internal/platform/providers/registry.ts`

- Added `_cachedModelRegistry: ModelDefinition[] | null` and `_modelRegistryRevision: number` fields.
- Added `_invalidateModelRegistry()` private method that nulls the cache and bumps the revision.
- `getModelRegistry()` returns the cached array immediately on cache hit; rebuilds via `buildModelRegistry()` only when null.
- Cache is invalidated in every state-mutating method: `register()`, `registerRuntimeProvider()` (both register and the returned unregister callback), `registerDiscoveredProviders()`, `updateCatalogState()`, `setModelContextCap()` (only when a match is found), and `loadCustomProviders()` (after swapping `customModels`).

### P2+P3: `recentEvents.unshift()` O(n) ring buffer replacement

**File**: `packages/sdk/src/_internal/platform/control-plane/gateway.ts`

- Replaced `private readonly recentEvents: ControlPlaneRecentEvent[]` with a 500-slot pre-allocated circular ring buffer (`_recentEventsRing`, `_recentEventsHead`, `_recentEventsCount`, `_recentEventsCapacity`).
- `rememberEvent` now writes at `_recentEventsHead` and advances the pointer modularly — O(1) regardless of ring size, no array shifting.
- A `private get recentEvents()` accessor reconstructs the ordered array on demand for `getSnapshot()` and `listRecentEvents()` callers, preserving the existing API contract.
- Constructor pre-allocates `new Array(500)` for the ring.

### P4: `rememberEvent` `syncControlPlaneState` debounce

**File**: `packages/sdk/src/_internal/platform/control-plane/gateway.ts`

- Added `_syncScheduled: boolean` flag and `_scheduleControlPlaneSync(lastEventAt)` private method.
- `_scheduleControlPlaneSync` is a no-op if a sync is already scheduled; otherwise sets the flag, calls `setImmediate`, and dispatches exactly one `syncControlPlaneState` per event-loop tick regardless of how many events arrived in that tick.
- `rememberEvent` now calls `_scheduleControlPlaneSync(record.createdAt)` instead of dispatching directly, coalescing N per-event Zustand reducer runs into 1 per tick during high-throughput streaming.

### P5: `ConversationManager.getMessagesForLLM()` memoization

**File**: `packages/sdk/src/_internal/platform/core/conversation.ts`

- Added `_messagesRevision: number`, `_cachedLLMMessages: ProviderMessage[] | null`, and `_cachedLLMRevision: number` fields.
- `getMessagesForLLM()` returns the cached array when `_cachedLLMRevision === _messagesRevision`; re-scans and re-caches otherwise.
- `_messagesRevision` is incremented in every message-mutating method: `addUserMessage`, `addAssistantMessage`, `addToolResults`, `addSystemMessage`, `undo`, `redo`, `removeMessagesAfter` (only when truncation happens), `markLastUserMessageCancelled` (only when a match is found), `startStreamingBlock`, `updateStreamingBlock` (only when content changes), `finalizeStreamingBlock`, `replaceMessagesForLLM`, `resetAll`, `switchBranch`, `mergeBranch`, and `fromJSON`.

## Migration

None. All changes are internal to private methods and fields; no public API surface changed.
