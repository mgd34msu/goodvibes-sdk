# 0.18.27

`0.18.27` adds the reusable template-manager surface needed for the next TUI cutover step.

The concrete gap was that the TUI still carried its own local `templates/manager.ts` implementation even after the surrounding session-content command surface had already moved onto SDK-owned exports. That left one more copied platform utility in the host tree and blocked the TUI from importing the template manager from the published package.

## What Changed

### New Public Runtime Surface

This release adds:

- `@pellux/goodvibes-sdk/platform/templates/manager`

That surface now exports:

- `TemplateManager`
- `TemplateManagerRoots`
- `TemplateEntry`
- `parseTemplateArgs(...)`

### Host-Owned Template Roots

The reusable implementation no longer hardcodes a TUI-owned template storage layout.

Instead, the SDK manager now accepts host-supplied roots:

- `projectRoot`
- `homeDirectory`
- optional `projectDirectory`
- optional `globalDirectory`

That keeps storage policy at the host boundary while keeping the actual template behavior in the SDK:

- save
- load
- list
- delete
- recursive template expansion
- positional and named argument parsing

### Downstream Effect

This patch is the release boundary needed for the TUI cutover:

- the TUI can now remove its local `src/templates/manager.ts`
- session-content command handling and template tests can import the published SDK path directly
- the template manager no longer needs to be carried as a copied host implementation

## Validation

This release was validated with:

- `bun run validate`
- pack check
- tarball install smoke
- the downstream TUI build/typecheck boundary, which now correctly fails on `0.18.26` and requires this new published export
