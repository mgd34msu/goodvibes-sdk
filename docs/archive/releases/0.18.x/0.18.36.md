# 0.18.36 — Daemon facade respects injected port and host

**Released**: 2026-04-16
**Type**: Patch (bug fix)

## Summary

Fixes a long-standing bug (predating the 0.18.34–0.18.35 hostMode/port churn) where `resolveDaemonFacadeRuntime` in `platform/daemon/facade-composition.ts` built the returned `port` and `host` entirely from `configManager` via `resolveHostBinding`, ignoring the `config.port` and `config.host` values passed directly to `new DaemonServer({ port, host, ... })`.

Result: tests that bind a random high port (e.g., 39421) and embedders that pass explicit host/port silently got `controlPlane.port: 3421` / `controlPlane.host: 127.0.0.1` from the configManager instead.

## Fix

`port` and `host` in the resolved runtime now prefer the constructor-injected values and fall back to the hostMode-aware binding. This matches the pattern `HttpListener` already uses:

```ts
this.port = config.port ?? resolvedHttpBinding.port;
this.host = config.host ?? resolvedHttpBinding.host;
```

## Impact

Restores 50+ daemon integration tests that pin a test port via the `DaemonServer` constructor. Unblocks embedders who depend on direct port/host injection.
