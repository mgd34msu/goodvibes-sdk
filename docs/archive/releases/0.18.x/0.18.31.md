# 0.18.31

`0.18.31` replaces the broken inline QR encoder in `platform/pairing/` with a vendored pure-TypeScript implementation that handles all QR versions and error correction levels correctly.

## What Changed

- Replaced the broken inline QR encoder with the vendored Nayuki QR Code generator (pure TypeScript, MIT license)
- No npm dependencies are added for QR generation — the vendored library is self-contained and handles all QR versions and error correction levels

## Why This Patch Exists

The original inline encoder shipped in `0.18.30` produced invalid QR codes for certain input lengths and error correction configurations. The Nayuki implementation is a well-tested, MIT-licensed pure-TypeScript library that covers the full QR Code specification without requiring any native bindings or npm dependencies.
