# 0.18.17

`0.18.17` is the next SDK host-boundary cleanup pass.

## What Changed

- Removed more baked-in `.goodvibes/goodvibes/...` storage paths from reusable SDK modules.
- Moved additional storage-root decisions to explicit host-owned inputs.
- Kept the canonical `goodvibes` root only in concrete daemon host wiring instead of reusable package internals.

## Updated Surfaces

- `ConfigManager`
  - now requires `surfaceRoot` when deriving config paths from `homeDir` / `workingDir`
- `SecretsManager`
  - now requires `surfaceRoot`
  - project/user secret store defaults now derive from the host-owned surface root
- `createRuntimeServices`
  - now passes explicit host-owned task-graph and distributed-runtime store paths
- `CrossSessionTaskRegistry`
  - now takes an explicit task-graph file path
- `DistributedRuntimeManager`
  - now takes an explicit store instance or store path
- MCP discovery and provider discovery persistence
  - now require host-owned `surfaceRoot`
- `KeybindingsManager`, guidance persistence, plugin state, and sandbox provisioning
  - now require explicit host-owned surface-root or file-path input when deriving paths

## Validation

- `bun x tsc -b packages/sdk --force --pretty false`
- `bun run validate`
